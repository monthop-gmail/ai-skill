# คู่มือความเข้าใจระดับการทำงานของ AI: Prompt vs. Skill vs. Workflow

ในปัจจุบัน การนำ AI มาใช้ในองค์กรไม่ได้มีแค่การ "พิมพ์คุย" (Chat) เท่านั้น แต่แบ่งออกเป็น 3 ระดับความซับซ้อนตามรูปแบบการทำงาน ดังนี้:

---

## 1. AI Prompt-based (เน้นการสั่งการ)

เป็นการใช้งานระดับพื้นฐานที่สุด (Direct Interaction) โดยอาศัยการเขียนคำสั่งที่แม่นยำ

* **นิยาม:** การส่งคำสั่ง (Prompt) เข้าไปเพื่อให้ LLM (เช่น ChatGPT, Gemini) ประมวลผลและตอบกลับ
* **หัวใจหลัก:** **Prompt Engineering** (การเขียนคำสั่งให้ฉลาด)
* **เปรียบเทียบ:** เหมือนเรา **"สั่งงานเลขาด้วยปากเปล่า"** เลขาจะทำงานได้ดีแค่ไหน ขึ้นอยู่กับว่าเราบรีฟงานละเอียดเพียงใด
* **ข้อดี:** ใช้งานได้ทันที, ยืดหยุ่นสูง
* **ข้อจำกัด:** AI ไม่มีเครื่องมือช่วย (ใช้วิธี "มโน" จากฐานข้อมูลที่มี), ทำงานต่อเนื่องไม่ได้

## 2. AI Skill-based (เน้นความสามารถเฉพาะทาง)

เป็นการยกระดับ AI ให้สามารถ "ลงมือทำ" (Action) ได้มากกว่าแค่การตอบคำถาม

* **นิยาม:** การเชื่อมต่อ AI เข้ากับเครื่องมือภายนอก (Tool Use / Function Calling)
* **หัวใจหลัก:** **Tools & API Integration** (การมอบเครื่องมือให้ AI)
* **เปรียบเทียบ:** เหมือนเรา **"มอบคอมพิวเตอร์และอินเทอร์เน็ตให้เลขา"** เลขาสามารถไปค้นหาข้อมูลจริงจาก Google หรือกดคำนวณเลขใน Excel ได้
* **ข้อดี:** ข้อมูลแม่นยำขึ้น ลดการหลอน (Hallucination), ทำงานเฉพาะทางได้จริง
* **ข้อจำกัด:** ต้องมีการตั้งค่าทางเทคนิคเพื่อเชื่อมต่อระบบ

## 3. AI Workflow-based (เน้นกระบวนการทำงาน)

ระดับสูงสุดคือการร้อยเรียง AI หลายตัวมาทำงานร่วมกันอย่างเป็นระบบอัตโนมัติ

* **นิยาม:** การวางลำดับขั้นตอน (Orchestration) ให้ Agent หลายตัวทำงานต่อกันจนจบโปรเจกต์
* **หัวใจหลัก:** **Agentic Workflow** (การวางแผนผังการทำงาน)
* **เปรียบเทียบ:** เหมือนเรา **"สร้างแผนกหรือระบบโรงงาน"** ที่มีเลขาหลายคน คนหนึ่งหาข้อมูล คนหนึ่งเขียน คนหนึ่งตรวจสอบงาน
* **ข้อดี:** จัดการงานที่ซับซ้อนได้โดยอัตโนมัติ (End-to-end), ทำงานซ้ำๆ ได้มาตรฐาน
* **ข้อจำกัด:** ออกแบบยากที่สุด ต้องใช้ Framework ช่วยจัดการ

---

## ตารางเปรียบเทียบสรุป

| หัวข้อ | Prompt-based | Skill-based | Workflow-based |
| --- | --- | --- | --- |
| **รูปแบบ** | สั่งงานด้วยข้อความ | มอบเครื่องมือเสริม | วางลำดับขั้นตอนงาน |
| **ความยาก** | ต่ำมาก | ปานกลาง | สูง |
| **จุดเด่น** | รวดเร็ว ยืดหยุ่น | ทำงานจริงได้แม่นยำ | อัตโนมัติและซับซ้อน |
| **สิ่งที่ต้องเน้น** | Prompt Engineering | Tool/API Integration | Business Logic & Sequencing |

---

## ตัวอย่างการประยุกต์ใช้ผ่าน "CrewAI" Framework

เพื่อให้เห็นภาพการนำไปเขียน Code หรือออกแบบระบบ นี่คือตัวอย่างใน Framework ยอดนิยมอย่าง CrewAI:

### 1. ระดับ Prompt (กำหนด Role)

เราใช้ Prompt เพื่อปั้นตัวตน (Agent) ขึ้นมา

```python
researcher = Agent(
  role='นักวิจัย',
  goal='สรุปเทรนด์เทคโนโลยีปี 2026',
  backstory='คุณเป็นนักวิเคราะห์ข้อมูลที่เก่งที่สุด'
)
```

### 2. ระดับ Skill (มอบ Tools)

เราใส่เครื่องมือ (Skill) เพื่อให้ Agent ออกไปทำงานนอกกรอบได้

```python
from crewai_tools import SerperDevTool

# มอบทักษะการค้นหา Google ให้กับ Agent
researcher.tools = [SerperDevTool()]
```

### 3. ระดับ Workflow (สร้าง Crew)

เรากำหนดลำดับงานว่าใครต้องทำอะไรก่อน-หลัง

```python
# วางขั้นตอน (Workflow): วิจัยก่อน -> แล้วค่อยให้คนเขียนสรุป
crew = Crew(
  agents=[researcher, writer],
  tasks=[research_task, writing_task],
  process=Process.sequential # ทำงานตามลำดับ 1-2
)
crew.kickoff()
```

---

## Key Takeaway

* หากต้องการคำตอบไวๆ ให้ใช้ **Prompt**
* หากต้องการให้ AI ทำงานร่วมกับข้อมูลบริษัท/อินเทอร์เน็ต ให้พัฒนา **Skill**
* หากต้องการสร้างระบบทำงานอัตโนมัติ (เช่น สรุปข่าวทุกเช้า, วิเคราะห์คู่แข่งรายสัปดาห์) ให้สร้าง **Workflow**
